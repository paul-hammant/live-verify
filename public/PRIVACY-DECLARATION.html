Privacy Declaration

Verific: OCR to hash verifier

TL;DR: Verific runs entirely in your browser. Your scanned documents never leave your device. The app only makes minimal HTTP GET requests to verify hashes against public certification servers.

What Verific Does

Verific is a browser-based application that scans text within registration marks, computes a cryptographic hash, and verifies it against a certification server. The entire OCR and hashing process happens locally in your browser.

Data Processing: 100% Local

All sensitive operations occur entirely within your browser:

  • Camera access: Uses browser MediaStream API - video stays on your device
  • OCR text extraction: Performed locally using Tesseract.js WebAssembly
  • Image processing: OpenCV.js processes images entirely in-browser
  • Text normalization: Character mapping happens client-side in JavaScript
  • SHA-256 hashing: Computed locally using browser's SubtleCrypto API
Your document images and extracted text NEVER leave your device. They are never uploaded, transmitted, or stored on any server.

Network Requests: What Gets Sent

Verific makes only two types of HTTP GET requests during the verification process:

Request Type Purpose Data Sent Example
Metadata Request Fetch OCR normalization rules for a domain None (simple GET) GET https://example.com/.verific-meta.json
Hash Verification Verify computed hash exists on certification server SHA-256 hash only (no document content) GET https://example.com/c/abc123...xyz

What is NOT sent:

  • Document images or photos
  • Extracted OCR text (raw or normalized)
  • Camera feed or video frames
  • Personal information or identifiers
  • Geolocation data
  • Device fingerprints
Important: The SHA-256 hash sent during verification is a cryptographic one-way function. It is mathematically infeasible to reverse-engineer the original document text from the hash. The hash alone reveals nothing about your document's content.

Third-Party Resources

Verific loads the following libraries from CDNs to provide functionality:

  • Tesseract.js (cdn.jsdelivr.net) - OCR engine for text extraction
  • OpenCV.js (docs.opencv.org) - Computer vision for image processing
  • PSL (Public Suffix List) (unpkg.com) - Domain parsing for security

These libraries are loaded over HTTPS and execute entirely within your browser's JavaScript sandbox. They do not transmit data back to their CDN providers.

What Verific Does NOT Do

  • Does NOT store scans: No images or text are saved, cached, or persisted
  • Does NOT upload documents: Your scanned content never leaves your browser
  • Does NOT track users: No analytics, cookies, or tracking scripts
  • Does NOT collect personal data: No registration, accounts, or PII collection
  • Does NOT use external APIs for OCR: All text extraction is local
  • Does NOT transmit to third parties: Only connects to user-specified verification domains

Data Retention

Verific has zero data retention:

  • Images exist only in browser memory during active scanning sessions
  • Extracted text is discarded when you close the browser tab
  • No localStorage, sessionStorage, IndexedDB, or cookies are used
  • Camera access is released when you stop the camera or close the tab

Browser Permissions

Verific requests the following browser permissions:

  • Camera access: Required to capture document images (you must explicitly grant this)
  • Clipboard write: Used when you click "Copy Hash" or "Copy Image" buttons (optional)

You can revoke these permissions at any time through your browser settings.

Open Source Transparency

Verific is fully open source under the AGPL-3.0-or-later license. You can:

  • Inspect the complete source code on GitHub
  • Verify that no data is being exfiltrated
  • Run your own instance on localhost or private servers
  • Audit the code to confirm this privacy declaration

Verification Server Privacy

When you scan a document, Verific connects to the domain specified in the document's verification URL (e.g., verify:example.com/path). The privacy practices of that certification server are:

  • Not controlled by Verific - Each organization runs their own server
  • May log HTTP requests - Standard web server behavior (IP address, timestamp, user agent)
  • Receive only the hash - No document content is ever transmitted

For questions about a specific verification server's privacy practices, contact the organization that issued the document.

Security Considerations

While Verific protects your privacy, users should be aware:

  • HTTPS required: Verification URLs are automatically upgraded to HTTPS to prevent interception
  • No server-side trust: Verific trusts the certification server specified in the document URL
  • Client-side only: Security depends on your browser's integrity and isolation
